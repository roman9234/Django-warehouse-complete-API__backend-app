---- Что сервис делает ----

Это бэкэнд-сервис, который симулирует работу маркетплейса.

Поставщик создаёт продукты.
Заказчик заказывает продукт на склад.
Поставщик помечает продук как доставленный.
Заказчик помечает продукт как полученный

Структуры данных:
- пользователь (requester / supplier)
- склад (в нём размещаются заказы)
- продукт (заказы на продукты размещаются)
- заказ (может быть доставлен поставщиком, может быть забран заказчиком)


Админы имеют доступ ко всем действиям
Только админы могут создавать склады

Только неавторизованные пользователи могут создавать аккаунты
При создании аккаунта пользователь выбирает, Suppliers он или Requester

Только Suppliers могут создавать продукты

Только Requester может создавать поля requests и редактировать поле request_retrieved
Только supplier может редактировать поле request_supplied
Создание request -> заказ доставляется (request_supplied) -> заказ забирается заказчиком
(request_retrieved)
